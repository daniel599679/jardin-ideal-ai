## DEFINICIÓN DE PRIORIDADES

```
P1 — GENERA INGRESOS (ejecutar esta semana)
  Definición: La idea tiene impacto DIRECTO en leads, ventas o retención de clientes.
  Ejemplos: nuevo tipo de ad que convierte mejor, automatización que captura leads, script de venta.
  Criterio: ¿Si no hacemos esto esta semana, perdemos dinero? → P1

P2 — MEJORA LA PRODUCCIÓN (ejecutar este mes)
  Definición: La idea hace que el equipo o los agentes produzcan más rápido o con mayor calidad.
  Ejemplos: mejora a FLOAT, nuevo módulo del sistema, automatización de contenido.
  Criterio: ¿Esto hace que el trabajo diario sea significativamente más eficiente? → P2

P3 — INFRAESTRUCTURA FUTURA (ejecutar este trimestre)
  Definición: La idea construye capacidad a largo plazo pero no tiene impacto inmediato en ingresos.
  Ejemplos: nuevo sistema, expansión a Groupe Ideal, integración de nueva herramienta.
  Criterio: ¿Esto paga dividendos en 3+ meses? → P3
```

## ÁRBOL DE PRIORIZACIÓN (usar para clasificar)

```
¿La idea genera leads o ventas directamente?
  SÍ → P1 → ejecutar esta semana
  NO → continuar

¿La idea hace que producción de contenido sea ≥30% más eficiente?
  SÍ → P1 o P2 dependiendo del impacto en revenue indirecto
  NO → continuar

¿La idea mejora un proceso existente?
  SÍ → P2 → ejecutar este mes
  NO → continuar

¿La idea es infraestructura para crecer en Q3+?
  SÍ → P3 → planificar para el trimestre
  NO → DESCARTAR o guardar en BACKLOG

¿Puede ejecutarse en <2 horas?
  SÍ + impacto claro → subir un nivel de prioridad (P3→P2, P2→P1)
  SÍ + impacto dudoso → mantener en su nivel
```
